## 📖 项目简介

**Review Helper** 是一款专为学术研究者设计的智能文献综述生成工具。它通过整合多个文献数据库（Google Scholar、烂番薯学术等）、AI关键词提取、智能文献筛选和自动综述生成等功能，将原本需要数周甚至数月的文献综述工作，压缩到几个小时即可完成。

### 🎯 我们为什么做这个？

**痛点：**
- 📚 **文献搜索耗时**：手动在Google Scholar上搜索和筛选相关文献，需要花费大量时间
- 🔍 **关键词提取困难**：如何从研究主题中提取有效的搜索关键词，往往需要反复尝试
- 📝 **文献筛选繁琐**：需要逐篇阅读文献摘要，判断是否与研究主题相关
- ✍️ **综述撰写耗时**：整理文献信息、归纳观点、撰写综述内容，需要大量人工工作
- 💾 **数据管理混乱**：文献信息、中间结果分散在各个地方，难以统一管理
- 🎭 **虚假文献问题**：大模型直接生成的综述往往存在虚假文献，引用不存在的文献或捏造文献信息
- 💰 **工具收费昂贵**：市面上现有的文献综述工具大多需要付费订阅，增加了研究成本

**解决方案：**
- 🤖 **AI驱动的自动化流程**：从关键词提取到综述生成，全程AI辅助
- 🔗 **多数据源集成**：无缝集成Google Scholar、烂番薯学术等多个文献数据库，直接搜索和提取学术文献信息，确保所有文献真实存在
- ✅ **基于真实文献的综述生成**：只使用从实际文献库搜索到的真实文献，杜绝虚假文献问题
- 📊 **智能项目管理**：所有数据集中管理，支持多项目并行工作
- 🎨 **简约大气的界面**：科研风格设计，专注于内容本身
- 🆓 **完全免费开源**：基于MIT许可证，无需付费订阅，降低研究成本

## ✨ 核心功能

### 模块一：文献查找工作流

一个完整的工作流包含4个节点，按顺序自动执行：

- **节点1：关键词分析** - AI自动从研究需求中提取搜索关键词
- **节点2：文献搜索** - 在Google Scholar、烂番薯学术等数据源中自动搜索文献
- **节点3：文献补全** - 自动为搜索结果补充完整摘要信息
- **节点4：精选文献** - AI智能筛选相关文献，支持手动选择和编辑

**特色功能**：
- 🤖 **AI驱动的自动化流程**：从关键词提取到文献筛选，全程AI辅助
- 🔗 **多数据源支持**：集成Google Scholar、烂番薯学术等多个文献库
- 📊 **灵活配置**：可设置文献数量、年份范围、语言偏好等参数
- ✏️ **完整编辑功能**：可编辑文献的所有信息（标题、作者、年份、来源、摘要等）
- 📤 **Excel导出**：支持导出为Excel格式，便于备份和分享

### 模块二：综述撰写工作流

基于已选定的真实文献，自动生成高质量的文献综述：

- **大纲生成**：AI根据研究需求自动生成文献综述大纲
- **基于真实文献**：只使用从Google Scholar等实际搜索到的真实文献，确保所有引用文献都真实存在，杜绝虚假文献问题
- **AI自动撰写**：基于选定的真实文献，AI自动生成结构清晰、逻辑严密的文献综述
- **准确引用**：自动在引用位置标记文献序号 `[文献序号]`，所有文献信息可追溯
- **可编辑内容**：支持对生成的综述进行编辑和优化
- **Word导出**：导出为Word兼容的HTML格式，便于进一步编辑

**特色功能**：
- 📝 **可定制大纲**：支持编辑和调整AI生成的大纲
- 🎯 **多章节支持**：可设置章节数量（1-10章）
- 🌐 **多语言支持**：支持中文和英文综述生成
- 📏 **字数控制**：可设置每章字数要求

### 项目管理

- **多项目支持**：可同时管理多个研究项目，每个项目独立配置
- **子项目管理**：每个项目可创建多个文献查找子项目和综述撰写子项目，便于管理不同的研究方向
- **自动保存**：所有操作自动保存，防止数据丢失
- **状态追踪**：清晰显示工作流节点进度（pending/active/completed）
- **数据持久化**：所有项目数据本地存储，支持离线使用
- **多API支持**：每个项目可独立配置不同的AI服务商（DeepSeek、Gemini、SiliconFlow、Poe）

## 🚀 快速开始

### 环境要求

- **Node.js** >= 14.0.0
- **npm** >= 6.0.0
- **AI API Key**（用于AI功能，支持以下服务商）：
  - [DeepSeek](https://www.deepseek.com/) - 推荐，性价比高
  - [Google Gemini](https://gemini.google.com/) - 功能强大
  - [SiliconFlow](https://siliconflow.cn/) - 国内服务
  - [Poe](https://poe.com/) - 多种模型支持

### 安装步骤

1. **克隆或下载项目**
   ```bash
   git clone <repository-url>
   cd 文献项目
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动应用**
   ```bash
   npm start
   ```

### 首次使用

1. **创建新项目**
   - 在主页面点击"新建项目"
   - 输入项目名称
   - （可选）输入项目简介作为提醒

2. **配置API Key**
   - 在项目详情页面选择API供应商（DeepSeek、Google Gemini、SiliconFlow、Poe）
   - 输入对应的API Key
   - API Key会自动保存到项目中，每个项目可独立配置

3. **开始使用**
   - 按照模块顺序完成文献搜索、整理和综述生成

## 📖 使用指南

### 模块一：文献查找工作流

工作流包含4个节点，按顺序执行：

#### 节点1：关键词分析
1. **输入文献查找需求**
   - 在"查询的文献综述需求"框中详细描述您的研究主题
   - 例如："我需要查找关于'共享自动驾驶汽车以及mass系统'的文献，重点关注技术实现、系统架构、安全性和用户体验等方面"

2. **配置搜索参数**
   - **文献来源库**：选择Google Scholar或烂番薯学术
   - **文献数量**：需要搜索的文献总数（默认50篇）
   - **年份限制**：设置近N年占比，例如"近5年占60%"
   - **文献语言**：选择中文或英文

3. **生成关键词**
   - 点击"生成关键词"按钮
   - AI会自动分析需求并提取相关搜索关键词
   - 可以查看和确认提取的关键词

#### 节点2：文献搜索
1. **自动搜索**
   - 系统会根据提取的关键词自动搜索文献
   - 支持Google Scholar、烂番薯学术等多个数据源
   - 首次使用Google Scholar需要完成验证

2. **查看搜索结果**
   - 查看按关键词分组的搜索结果
   - 可以编辑、删除搜索结果中的文献

#### 节点3：文献补全
1. **自动补全摘要**
   - 系统会自动为搜索结果中的文献补充完整摘要信息
   - 确保每篇文献都有完整的元数据

#### 节点4：精选文献
1. **AI智能筛选**
   - 点击"开始AI筛选"按钮
   - AI会分析每篇文献，判断是否与研究相关
   - 筛选结果会显示推荐理由

2. **手动选择**
   - 可以手动勾选或取消勾选文献
   - 支持全选、全不选等操作

3. **编辑和管理**
   - 可以编辑文献信息
   - 可以删除不需要的文献
   - 支持导出为Excel进行备份

> **注意**：文献整理功能已整合到"模块一：文献查找工作流"的节点4中，无需单独操作。

### 模块二：综述撰写工作流

1. **创建综述撰写子项目**
   - 在项目详情页面点击"新建综述撰写子项目"
   - 系统会提示选择关联的文献查找子项目（作为基础文献资料）

2. **配置项目**
   - **API供应商**：选择要使用的AI服务（DeepSeek、Gemini等）
   - **大纲要求**（可选）：描述您对文献综述大纲的要求
   - **章节数**：设置文献综述的章节数量（默认3章）
   - **输出语言**：选择中文或英文

3. **生成大纲**
   - 点击"生成大纲"按钮
   - AI会根据需求自动生成文献综述大纲
   - 可以编辑和调整大纲内容
   - 点击"保存"确认大纲

4. **生成综述**
   - 查看已关联的文献列表（来自文献查找子项目）
   - **综述要求**（可选）：填写对综述的具体要求
   - **每章字数要求**（可选）：指定每章的字数范围
   - 点击"生成综述"按钮
   - AI会基于选定的真实文献自动生成结构清晰、逻辑严密的文献综述
   - 所有引用都会标记文献序号 `[文献序号]`，确保可追溯

5. **编辑和导出**
   - 可以直接在文本框中编辑综述内容
   - 所有修改会自动保存
   - 点击"复制综述内容"复制到剪贴板
   - 点击"导出Word"导出为Word兼容的HTML文件

## 💡 使用技巧

### 提高文献查找效率

1. **需求描述要详细**：在节点1输入需求时，尽量详细描述研究主题、关注重点等，有助于AI提取更准确的关键词
2. **合理配置参数**：
   - **文献数量**：根据研究范围设置合适的文献总数（建议30-100篇）
   - **年份限制**：合理设置近N年占比，平衡文献的新颖性和全面性
   - **文献来源**：根据研究领域选择合适的数据库（国内用户可选烂番薯学术，英文研究选Google Scholar）
3. **关键词优化**：如果AI提取的关键词不够准确，可以在节点1查看并确认关键词
4. **利用一键查找**：配置完成后，可以使用"一键查找"功能自动完成所有节点

### 优化文献筛选结果

1. **AI筛选后手动检查**：节点4的AI筛选结果仅供参考，建议手动检查每篇文献的相关性
2. **编辑文献信息**：如果文献信息不完整或错误，可以点击编辑按钮手动补充和修正
3. **删除无关文献**：及时删除不相关的文献，保持列表整洁，提高后续综述质量
4. **批量操作**：支持全选、全不选、仅选AI推荐等批量操作，提高效率

### 改善综述质量

1. **选择高质量文献**：在节点4精选文献时，优先选择引用数高、年份较新、与研究主题高度相关的文献
2. **优化大纲结构**：在生成综述前，仔细检查和调整AI生成的大纲，确保结构合理
3. **设置字数要求**：根据实际需求设置每章字数要求，控制综述的详细程度
4. **编辑AI生成的综述**：AI生成的综述是基础，建议根据实际需求进行编辑和优化
5. **多次生成对比**：如果对结果不满意，可以删除后重新生成，或调整参数后重新生成

## ⚠️ 注意事项

1. **API Key安全**：请妥善保管您的API Key，不要泄露给他人。每个项目可独立配置不同的API供应商和Key
2. **Google Scholar验证**：首次使用Google Scholar搜索需要登录并完成验证，后续搜索会自动进行
3. **数据备份**：建议定期导出Excel文件进行备份，项目数据存储在`projects`文件夹中
4. **网络连接**：需要网络连接才能使用AI功能和文献搜索功能
5. **真实文献保证**：系统只使用从Google Scholar等实际搜索到的真实文献，杜绝虚假文献问题
6. **多项目支持**：可以同时管理多个研究项目，每个项目的数据独立存储
7. **子项目管理**：每个项目可以创建多个文献查找子项目和综述撰写子项目，便于管理不同的研究方向
